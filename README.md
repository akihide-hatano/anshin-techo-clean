# 📘 About Anshin-Techo

**「安心手帳」**は、高齢者の服薬記録を家族が**遠隔から管理**できるWebアプリケーションです。
Laravelの**メール送信機能**と**カレンダーUI**を活用し、服薬状況を**リアルタイムで確認・通知**します。

---

# ✨ 主な機能

- ✅ **内服記録の管理**
  ダッシュボードで服薬履歴を一括管理。

- 💊 **内服一覧**
  内服薬の一覧を検索機能と作用と副作用なども確認が可能。

- 📅 **服薬カレンダー表示**
  `FullCalendar.js` を使用し、服薬スケジュールを視覚的に表示。

- ✉️ **内服忘れ通知メール**
  服薬記録が「未完了」で登録されると、ユーザープロフィールの通知用メールアドレスへ自動通知。
  イベント／リスナー方式で非同期送信を実装し、件名・本文に必要情報を動的挿入。
  SMTP設定とキュー実行が必要。詳細は[開発者向け資料](docs/MAIL_NOTIFICATION.md)を参照。

---

## 🎯 なぜ作るのか（Problem & Goal）

**対象:** 自宅で服薬する高齢者と、その家族・介護者 
**現状の課題:** 
- 服薬の「抜け・漏れ」を家族が**その場で気づけない**
- 紙・口頭ベースで**記録が分散**し、継続率が低い
- 飲み忘れや重複服用が**可視化されない**

**提供価値（解決）:**
- カレンダーと通知で**当日の未服薬を即検知**
- 家族が**遠隔で状況を共有**（安心感の可視化）
- 継続しやすいUIで**記録の継続率**を高める

---

## 🩺 開発背景

私自身、看護師として働いていた経験から、入院中は看護師が服薬を管理しますが、
退院後は患者さん自身やご家族が管理しなければならない現状を何度も目にしました。
その中で、服薬管理が続けられず飲み忘れが発生し、再入院や症状悪化につながるケースもあります。

既存の服薬管理アプリは多機能すぎて使いづらい場合が多く
必要最低限の機能に絞り、誰でも簡単に継続できる形を目指しました。

---

## 🎯 ゴール

- 服薬管理に対する本人の不安を軽減する
- 家族が離れていても安心して見守れる状態を作る
- 「簡単・続けやすい」を最優先したUI/UXを実現する

## 📈 成功指標（KPIs）
> 以下は本アプリの開発者が設定した、運用を通じて達成を目指す目標値です。
> 実際の数値はユーザー環境や利用状況によって変動します。

- **アドヒアランス（服薬遵守率）:**
  `実際に飲んだ回数 ÷ 予定回数` を月次で可視化
  - 目標: **+5%以上改善**（例：80% → 85%）を3か月以内に達成
- **継続利用率（D30留存）:**
  初回記録から30日後のアクティブ率 **40%以上**  
- **家族との会話時間の向上:**
  服薬や体調に関する会話が**週あたり+10分以上**増加
  （利用者アンケートで測定）


# 🧱 技術スタックと選定理由

| 技術 | 用途 | 理由 |
|------|------|------|
| **Laravel 12** | バックエンド | RESTfulな構造と、Mail・Schedule機能の柔軟性が高い。 |
| **Blade + JavaScript** | フロントエンド | Laravelと親和性が高く、最小限の構成で動的UIを実現。 |
| **Vite** | ビルドツール | モダンな開発環境を即時反映で実現。 |
| **Tailwind CSS** | UIスタイリング | コンポーネントを素早く整形でき、保守性が高い。 |
| **FullCalendar.js** | カレンダー表示 | シンプルかつ高機能な日付UIライブラリ。 |
| **PostgreSQL** | RDBMS | JSONB対応・正規化のしやすさ。 |
| **pgAdmin** | DB管理ツール | GUIでのデータ確認やクエリ発行が容易。 |
| **Docker + Laravel Sail** | 環境構築 | 環境差異をなくし、開発効率を向上。 |
| **Mailpit** | メールテスト | 実際の送信をせずにローカル確認が可能。 |

# ⚙️ 開発環境セットアップ

```bash
# リポジトリをクローン
git clone https://github.com/your-username/anshin-techo-clean.git
cd anshin-techo-clean
```

```bash
# Dockerコンテナを起動
./vendor/bin/sail up -d
```

```bash
# 依存関係をインストール
sail composer install
sail npm install
```

```bash
# 環境ファイルの準備
cp .env.example .env
```

```bash

sail artisan migrate:fresh --seed
```

```bash

```

# 📝 現状の課題と今後の展望

現在の「安心手帳」は、ユーザーが自由に内服薬をCRUD操作できる仕組みですが、
これは柔軟性がある反面、**医療的な正確性や安全性の担保**という面では限界があります。

### ❗️ 現状の課題

- **薬剤の情報に公式APIが存在しない**
  → 薬の用法・用量・成分・商品名などを一貫して取得できるデータベースがない。

- **医療者でないと判断が難しい項目が多い**
  → 同じ薬でも用法や適応疾患が違ったり、服用タイミングが医師の裁量で変わることも。

- **誰でも薬データを登録・変更できてしまう**
  → 誤登録による副作用や服薬ミスのリスク。

---

### 🌱 今後の展望（将来的な拡張可能性）

- [ ] **公的・民間の薬剤データベースとの連携（API）**
      例：PMDA、KEGG、MediBank などとのデータ連携

- [ ] **薬剤師・医師アカウントの導入**
      医療従事者が服薬データを監修・承認できるようにする

- [ ] **薬ごとのプロファイル自動生成**
      「この薬は朝食後に1錠」などをプリセットで表示

- [ ] **監査ログの導入**
      「誰が・いつ・どの薬情報を変更したか」を記録
